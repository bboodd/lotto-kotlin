로또 프로그램 만들기

필수사항
8장까지 다 읽고 진행 권장
순수 코틀린을 위한 라이브러리를 제외한 다른 라이브러리나 프레임워크는 꼭 필요한게 아니면 지양
순수 코틀린 JVM 프로젝트로 작성
코틀린 버전은 2.2.20이상
Ktlint를 설치하고 엄격하게 코드 포멧팅을 지키기
AI도움 받지 않고 순수 손으로 개발 및 궁금한 사항도 인터넷 검색을 쓸것
로또는 하나에 1000원
로또 번호는 한 장에 6개
로또 숫자는 1~45내 숫자 중복없이
각 회차별로 당첨 번호는 랜덤으로 정해야하며 각 회차의 당첨번호는 그 회차의 모든 로또 구매 완료 이후 결정되어야함
당첨번호는 6개 + 보너스 번호 1개로 이루어짐
돈을 입금해서 해당 한도 내에서 로또는 무제한 구매 가능
예사) 10만원이 있을때 항상 10만원 어치를 구매하진 않을수도 있음 100장 이내로 몇개든 나눠서 구매 가능
당첨금은 다시 입금내역으로 들어감
1등 21억, 2등 6천만, 3등 150만, 4등 5만, 3등 5천
1등 당첨번호 6개 일치, 2등 5개 일치 + 보너스 번호 일치, 3등 5개 일치, 4등 4개일치, 5등 3개 일치
로또 한장을 뽑는 방법은 수동, 자동을 지원하며 수동, 자동 동시 구매 가능
예시) 유저가 한번에 총 20장을 구매할때 3장은 수동으로 입력, 17장은 자동으로 구매 할수 있어야함
회차는 시간으로 진행되어도 좋고 수동으로 개발자가 1회차씩 올리는것도 OK
각 회차의 결과는 저장되어야하며 프로그램을 껐다 켜면 이력을 자동으로 불러와서 다음 회차를 진행해야함
과거 회차의 결과를 조회할수 있는 기능이 있어야함
Validation을 신경을 잘써야 하며 에러 처리도 잘해야함
매직넘버의 사용은 지양하며 되도록 상수로 정의해야함
일급 컬렉션을 안쓰고 일단 작성해보고, 그걸 일급컬렉션 쓰는 코드로 리팩토링 해볼것
한개의 메소드가 20줄이 넘지 않도록 작성해야 함
로또의 당첨확률이 매우 낮다는 사실을 직접 느껴보고 로또의 덧없음을 느낄것

선택사항
로또 한장 번호 뽑는 방식에 반자동도 추가해볼것 (6개의 번호중에서 N개는 수동 6-N개는 자동)
로또 한장 번호를 뽑는 방식을 전략패턴으로 각 전략을 갈아낄수 있는 방식으로 작성해볼것
리드미도 잘 작성해볼것
각 클래스는 명확히 구분되는 1개의 책임만을 가져야 하며 한개의 메소드는 한개의 역할만 수행하도록 작업
